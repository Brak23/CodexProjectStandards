# Terraform profile

- Pin Terraform and provider versions.
- Commit dependency lockfiles.
- Use remote encrypted state with locking.
- Separate environments through explicit state and identity boundaries.
- Run format, validate, security policy, and plan in PRs.
- Require approval for apply.
- Use OIDC and least-privilege cloud roles.
- Treat state as sensitive.
- Never allow an agent to approve its own IAM expansion or production apply.
