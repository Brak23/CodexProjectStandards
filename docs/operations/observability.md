# Observability

Instrument:

- Structured logs
- Metrics
- Distributed traces
- Correlation identifiers
- Deployment version and release markers
- Business success and failure events
- Feature flag state when diagnostically useful

Monitor latency, traffic, errors, and saturation. Define SLIs and SLOs before general availability. Alerts must correspond to actionable user impact or error-budget risk, not merely unusual infrastructure graphs.

Document dashboard links, standard queries, alert ownership, and expected behavior during dependency failure.
