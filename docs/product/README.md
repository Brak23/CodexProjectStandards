# Product documentation

Product documentation defines what the system should accomplish and why. It is the source for feature briefs and acceptance criteria.

- [`vision.md`](vision.md): Users, problems, outcomes, non-goals, and success measures.
- [`domain-model.md`](domain-model.md): Core entities, invariants, lifecycle, and ownership.
- [`glossary.md`](glossary.md): Canonical terminology.

Keep implementation choices out of product documents unless they are actual product constraints.
