# Code review behavior evaluations

`scenarios.json` defines portable behavior contracts for the project-local code-review skill. The repository validator checks structure only; it does not invoke or certify a model.

A model-specific harness should supply a representative repository fixture and assert both required and prohibited behavior. Evaluate at least:

- target and snapshot handling,
- scope and authority,
- evidence requirement composition,
- dynamic dimension and seam expansion,
- binding-finding acceptance criteria,
- dispute and closure authority,
- ledger reconciliation and regression,
- aggregation without scalar collapse,
- deterministic gate behavior,
- review-ref storage and merge aliases.
