# Incident response

## Declare

Record severity, start time, affected users, systems, current symptoms, incident commander, technical lead, communications lead, and current release.

## Stabilize

Prioritize user safety and service restoration over root-cause certainty. Use approved rollback, disablement, traffic reduction, or dependency isolation. Preserve evidence and record every consequential action and time.

## Communicate

Provide regular factual updates. Separate confirmed facts, active hypotheses, actions, and next update time. Never expose sensitive data in public status communications.

## Investigate

Use logs, metrics, traces, deployment history, configuration, and data evidence. Identify the first incorrect state and causal chain.

## Recover

Verify critical user journeys, data integrity, error rates, latency, saturation, queues, and external dependencies. Define enhanced monitoring before ending the incident.

## Follow up

Create a blameless postmortem, assign corrective actions, update runbooks, and add regression protection.
