# Planning contract

A plan is an immutable manifest over approved intent, active decision revisions, stable obligations, current execution attempts, release milestone revisions, controlled evidence requirements, and generated graphs.

Plan approval and implementation authorization are separate. The plan may be structurally valid and human-approved while implementation remains prohibited.

The plan uses size classes only: `XS`, `S`, `M`, `L`, and `XL`. Every size class names the unknowns that could change it. Calendar estimates and person-day commitments are outside the planner contract.

Every active AC maps to at least one obligation and exactly one behavioral milestone. Every active obligation has one current execution. Every current execution belongs to one current milestone.
