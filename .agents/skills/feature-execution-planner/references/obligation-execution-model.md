# Obligation and execution model

`TASK-004` is the stable obligation: what approved intent requires to become true. `TASK-004/1` and `TASK-004/2` are immutable attempts to satisfy it.

Traceability and AC coverage follow the obligation. Code, evidence, assignee, decision basis, and implementation scope follow the execution attempt.

Historical attempts never disappear. A displaced attempt names its successor. A released attempt remains `done`; a correction is a new attempt that references the release it corrects.

Task accountability routes closure of the obligation. CODEOWNERS on changed source paths independently routes code approval.
